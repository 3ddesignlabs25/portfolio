Broken Python Virtual Environment (Raspberry Pi)

Problem

A Python application failed to start after a system update, reporting missing modules despite being previously functional.

Symptoms

  • ImportError messages at launch
  • Application worked on a different machine
  • No recent code changes

Diagnosis

  • Checked active Python version
  • Verified virtual environment path
  • Reviewed installed packages inside the environment

Resolution

Recreated the virtual environment, reinstalled dependencies from a requirements file, and verified execution using a clean shell session.

Lesson Learned

System updates can invalidate environments; maintaining reproducible setups prevents prolonged downtime.
